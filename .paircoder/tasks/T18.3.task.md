# Objective

Embed org_id and role in JWT claims so every authenticated request carries its authorization context without a DB lookup.

# Implementation Plan

1. Update create_access_token() in auth.py to accept optional org_id and role params
2. Add org_id and role to JWT payload
3. Update issue_token_for() to look up user's org_id and role, pass to create_access_token
4. Update decode_token() to extract and return org_id and role from payload
5. Update verify_auth() to return structured identity with org_id and role from JWT
6. Update callers: user_routes.py login, auth_routes.py token/refresh
7. Update tests

# Acceptance Criteria

- [x] JWT payload includes org_id and role claims
- [x] decode_token returns org_id and role
- [x] verify_auth returns structured identity (sub, org_id, role)
- [x] Token refresh preserves org_id and role
- [x] Tests pass
- [x] 0 ruff issues
