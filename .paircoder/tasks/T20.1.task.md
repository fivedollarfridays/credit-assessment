# Objective

Persist Stripe subscriptions in DB. Connect rate limiting to subscription tiers so paid customers get higher limits.

# Implementation Plan

1. Migrate _subscriptions dict to SubscriptionRepository
2. Update billing.py: update_subscription, get_subscription, list_subscriptions, count_active
3. Update handle_webhook to handle subscription.updated and subscription.deleted events
4. Replace hardcoded @limiter.limit("30/minute") with dynamic tier-based limiting:
   - Resolve user's subscription tier from auth identity
   - Apply TIER_LIMITS from rate_limit.py (FREE=10, STARTER=60, PRO=300, ENTERPRISE=unlimited)
5. Call record_usage() from _run_assessment() after successful assessment (gap-13)
6. Update tests

# Acceptance Criteria

- [x] Subscriptions persist across restarts
- [x] Rate limits vary by subscription tier
