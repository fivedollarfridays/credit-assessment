# Objective

Fix 3 MEDIUM-severity findings: unbounded user_id input, SQLite in production, and unvalidated billing URLs.

# Findings

| ID | File | Issue |
|----|------|-------|
| A04-1 | data_rights_routes.py:47-66 | No `max_length` on `user_id` query parameter |
| A04-3 | config.py:23 | SQLite `database_url` not rejected in production |
| A04-2 | billing.py:70-84 | `success_url`/`cancel_url` forwarded to Stripe unvalidated |

# Implementation Plan

## A04-1: user_id max_length
1. Add `max_length=255` to all `user_id` Query parameters in `data_rights_routes.py`
2. Test: verify 422 on oversized user_id

## A04-3: SQLite rejection in production
1. In `config.py`, add validator that raises if `is_production=True` and `database_url` starts with `sqlite`
2. Test: verify ValueError raised for sqlite URL in production mode

## A04-2: Billing URL validation
1. In `billing.py`, validate `success_url` and `cancel_url` against allowed domains or URL patterns
2. Only allow `https://` URLs from configured allowed origins
3. Test: verify rejection of arbitrary URLs, acceptance of valid ones
