# Objective

Formalize environment variable management with pydantic-settings for typed config and dotenv for local development.

| `src/modules/credit/tests/test_config.py` | New: test config loading and defaults |

# Implementation Plan

1. Add `pydantic-settings>=2.0` to pyproject.toml
2. Refactor `config.py` to use a `Settings` class:
   - `api_key: str | None = None`
   - `cors_origins: list[str] = ["http://localhost:3000"]`
   - `environment: str = "development"`
   - `log_level: str = "INFO"`
   - `host: str = "0.0.0.0"`
   - `port: int = 8000`
3. Create `.env.example` documenting all variables
4. Update `.gitignore` to exclude .env files
5. Update `router.py` to use Settings instead of standalone functions
6. Write tests for config loading, defaults, and validation
