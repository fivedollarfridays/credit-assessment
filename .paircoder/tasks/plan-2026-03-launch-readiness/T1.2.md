## Objective

Add `Field` constraints and a model validator to `AccountSummary` and `CreditProfile` in `types.py` so that `test_validation.py` passes (16 currently failing tests).

## Changes to `AccountSummary`

All numeric fields need `ge=0` constraints:

| Field | Current | Required |
|-------|---------|----------|
| `total_accounts` | `int` | `int = Field(ge=0)` |
| `open_accounts` | `int` | `int = Field(ge=0)` |
| `closed_accounts` | `int = 0` | `int = Field(default=0, ge=0)` |
| `negative_accounts` | `int = 0` | `int = Field(default=0, ge=0)` |
| `collection_accounts` | `int = 0` | `int = Field(default=0, ge=0)` |
| `total_balance` | `float = 0.0` | `float = Field(default=0.0, ge=0)` |
| `total_credit_limit` | `float = 0.0` | `float = Field(default=0.0, ge=0)` |
| `monthly_payments` | `float = 0.0` | `float = Field(default=0.0, ge=0)` |

## Changes to `CreditProfile`

| Field | Current | Required |
|-------|---------|----------|
| `overall_utilization` | `float` | `float = Field(ge=0, le=100)` |
| `payment_history_pct` | `float` | `float = Field(ge=0, le=100)` |
| `average_account_age_months` | `int` | `int = Field(ge=0, le=1200)` |
| `negative_items` | `list[str] = []` | `list[Annotated[str, Field(max_length=200)]] = Field(default=[], max_length=50)` |

Add a `@model_validator(mode="after")` that checks `score_band` matches the expected band for `current_score` based on `SCORE_BANDS`, raising `ValueError` with "score_band" in the message on mismatch.
