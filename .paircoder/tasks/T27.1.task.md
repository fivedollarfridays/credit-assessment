# Objective

Fix 3 HIGH-severity OWASP findings: CORS missing PATCH method, Sentry PII leakage, and timing oracle on inactive users.

# Findings

| ID | File | Issue |
|----|------|-------|
| A05-1 | router.py:99 | CORS `allow_methods` missing PATCH — breaks `PATCH /disputes/{id}/status` for browser clients |
| A02-1 | sentry.py:17-21 | Sentry initialized without `send_default_pii=False` — leaks JWT, API keys, credit data |
| A07-1 | user_routes.py:123-124 | Inactive-user check skips bcrypt, enabling timing-based account enumeration |

# Implementation Plan

## A05-1: CORS PATCH method
1. In `router.py`, add `"PATCH"` to `allow_methods` list in CORSMiddleware config
2. Test: verify PATCH is in allowed methods via OPTIONS preflight

## A02-1: Sentry PII protection
1. In `sentry.py`, add `send_default_pii=False` to `sentry_sdk.init()`
2. Test: verify init is called with `send_default_pii=False`

## A07-1: Timing oracle fix
1. In `user_routes.py`, always run `bcrypt.checkpw()` even for inactive users (use dummy hash)
2. Return same error for inactive and wrong-password cases
3. Test: verify bcrypt is always called regardless of `is_active` status
