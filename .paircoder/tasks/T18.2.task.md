# Objective

Connect the scoped API key system (admin_routes.py) into verify_auth() so per-org, per-role, expiring API keys are actually enforced on requests.

# Implementation Plan

1. Migrate _api_keys OrderedDict to ApiKeyRepository (DB persistence)
2. Update verify_auth() in assess_routes.py:
   - After JWT check fails, try lookup_api_key() before static settings.api_key
   - On match: return identity with org_id and role from the key
   - Check expiration, reject expired keys
3. Return structured auth identity (not just string) — include org_id and role
4. Keep settings.api_key as fallback for backward compatibility
5. Update tests for new auth flow

# Acceptance Criteria

- [ ] verify_auth() checks scoped API keys via ApiKeyRepository
- [ ] Expired API keys are rejected with 401
- [ ] API key identity includes org_id and role
- [ ] Static settings.api_key still works as fallback
- [ ] _api_keys in-memory dict removed
- [ ] Tests pass
- [ ] 0 ruff issues
