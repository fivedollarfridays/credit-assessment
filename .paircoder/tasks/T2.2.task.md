# Objective

Set up a GitHub Actions CI/CD pipeline that lints, tests, and builds the Docker image on every push and PR.

# Files to Create

| File | Purpose |
|------|---------|
| `.github/workflows/ci.yml` | Main CI pipeline (lint, test, build) |
| `.github/workflows/docker-publish.yml` | Build + push Docker image on release tags |

# Implementation Plan

1. Create `.github/workflows/ci.yml`:
   - **Trigger:** push to main, PRs to main
   - **Jobs:**
     - `lint`: ruff check + ruff format --check
     - `test`: pytest with coverage, fail if < 100%
     - `build`: docker build (verify image builds)
   - **Matrix:** Python 3.13
2. Create `.github/workflows/docker-publish.yml`:
   - **Trigger:** tags matching `v*`
   - **Jobs:** build + push to GitHub Container Registry (ghcr.io)
3. Add `ruff` to dev dependencies in pyproject.toml if not present

# Acceptance Criteria

- [x] CI runs on push to main and on PRs
- [x] Lint step catches formatting issues
- [x] Test step runs all tests with 100% coverage gate
- [x] Docker build step verifies the image builds
- [x] Docker publish workflow builds and tags on version releases
- [x] Pipeline completes in < 5 minutes
- [x] Tests pass locally (`pytest`)
