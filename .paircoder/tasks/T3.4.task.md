# Objective

Add Prometheus-compatible metrics (request latency, error rates, assessment counts) and enhanced health/readiness probes.

# Implementation Plan

1. Add `prometheus-fastapi-instrumentator` to dependencies
2. Instrument FastAPI app with default HTTP metrics
3. Add custom metrics: assessment_count, assessment_latency, error_rate
4. Enhance `/health` to include readiness checks (DB connectivity when DB exists)
5. Add `/metrics` endpoint for Prometheus scraping
6. Add `/ready` endpoint for Kubernetes readiness probe
7. Write tests for metrics endpoint and health check responses

# Acceptance Criteria

- [ ] GET /metrics returns Prometheus-format metrics
- [ ] Request latency histograms tracked per endpoint
- [ ] Error rates tracked by status code
- [ ] GET /health returns service health
- [ ] GET /ready returns readiness status
- [ ] Tests pass locally (`pytest`)
