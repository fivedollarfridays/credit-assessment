# Objective

Fix two MEDIUM-severity findings: CSP `'unsafe-inline'` defeats XSS protection (A05-1) and webhook secrets stored in plaintext (A02-2).

# Findings

| ID | Severity | File | Issue |
|----|----------|------|-------|
| A05-1 | MEDIUM | dashboard_routes.py:83 | CSP `script-src 'unsafe-inline'` defeats XSS protection |
| A02-2 | MEDIUM | models_db.py:138 | Webhook secrets stored in plaintext column |

# Implementation Plan

## A05-1: Dashboard CSP Hardening

**Current:** `_DASHBOARD_CSP` uses `'unsafe-inline'` for both script-src and style-src.
**Fix:** Remove `'unsafe-inline'` from `script-src`. Keep for `style-src` only (inline styles pose lower XSS risk). If dashboard HTML has inline `<script>` tags, extract to external files or use hash-based CSP.

- `dashboard_routes.py`: Update `_DASHBOARD_CSP` — remove `'unsafe-inline'` from script-src
- Check dashboard HTML for inline scripts; refactor if needed
- Tests: Update CSP assertion tests

## A02-2: Webhook Secret Encryption

**Current:** `secret` column is plaintext `String(255)`.
**Fix:** Add application-level envelope encryption using `cryptography.fernet`. Create `_encrypt_secret()` / `_decrypt_secret()` helpers in a new `crypto.py` module. Key derived from `WEBHOOK_ENCRYPTION_KEY` env var.

- New `src/modules/credit/crypto.py`: `encrypt_field()` and `decrypt_field()` using Fernet
- `config.py`: Add `webhook_encryption_key: str | None = None` setting
- `webhook_routes.py`: Encrypt secret before storage
- `webhook_delivery.py`: Decrypt secret when computing HMAC signatures
- `models_db.py`: Update docstring to reflect application-level encryption
- Tests: Encrypt/decrypt round-trip, missing key fallback (plaintext backward compat)
