# Objective

Log every assessment request and response for compliance auditing: who requested it, when, what input was provided, what result was returned.

# Implementation Plan

1. Create AuditLog database model: request_id, user_id, org_id, timestamp, action, request_body_hash, response_summary
2. Add audit middleware that logs every /assess call
3. Create admin endpoint to query audit logs with filters
4. Implement log retention policy (configurable, default 7 years for FCRA)
5. Ensure PII is handled appropriately (hash sensitive fields)
6. Write tests for audit log creation and querying

# Acceptance Criteria

- [ ] Every /assess call creates an audit log entry
- [ ] Audit logs include: who, when, what, result summary
- [ ] Admin endpoint for querying audit logs
- [ ] PII fields hashed in audit records
- [ ] Retention policy enforced
- [ ] Tests pass locally (`pytest`)
