# Objective

Add API versioning with /v1/ prefix for backwards compatibility as the API evolves.

# Implementation Plan

1. Create versioned router: `APIRouter(prefix="/v1")`
2. Move /assess and /auth endpoints under /v1/
3. Keep /health and /metrics at root (unversioned)
4. Add deprecation headers for unversioned endpoint access
5. Update OpenAPI docs to reflect versioned paths
6. Write tests for versioned endpoint access

# Acceptance Criteria

- [ ] POST /v1/assess works as primary endpoint
- [ ] GET /health remains at root (unversioned)
- [ ] OpenAPI spec reflects /v1/ paths
- [ ] Deprecation header on legacy unversioned access
- [ ] Tests pass locally (`pytest`)
