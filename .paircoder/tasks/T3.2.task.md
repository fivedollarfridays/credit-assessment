# Objective

Replace the single API key auth with JWT-based authentication supporting multiple users, key rotation, and stateless token validation.

# Implementation Plan

1. Add `python-jose[cryptography]` and `passlib[bcrypt]` to dependencies
2. Create `src/modules/credit/auth.py`: JWT creation, validation, key rotation
3. Create token endpoints: POST /auth/token (issue), POST /auth/refresh
4. Update router.py: replace API key dependency with JWT validation
5. Support both API key (deprecated) and JWT during migration period
6. Add JWT-related settings to config.py (secret, algorithm, expiry)
7. Write tests for token lifecycle, expiry, invalid tokens, key rotation

# Acceptance Criteria

- [ ] JWT tokens issued via POST /auth/token with credentials
- [ ] Protected endpoints validate JWT Bearer tokens
- [ ] Token refresh works before expiry
- [ ] Invalid/expired tokens return 401
- [ ] Key rotation supported without downtime
- [ ] Legacy API key still works (deprecated, logged)
- [ ] Tests pass locally (`pytest`)
