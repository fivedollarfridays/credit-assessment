# Objective

Implement a webhook system to notify customers of async events (assessment completed, subscription changes, rate limit warnings).

# Implementation Plan

1. Create webhook registration endpoint: POST /v1/webhooks
2. Define event types: assessment.completed, subscription.updated, rate_limit.warning
3. Implement delivery with retry logic (exponential backoff, max 5 attempts)
4. Add HMAC signature verification for webhook payloads
5. Create webhook delivery log for debugging
6. Write tests for registration, delivery, retry, and signature verification

# Acceptance Criteria

- [x] Customers can register webhook URLs
- [x] Events delivered with HMAC signatures
- [x] Failed deliveries retried with exponential backoff
- [x] Delivery log accessible to customers
- [x] Tests pass locally (`pytest`)
