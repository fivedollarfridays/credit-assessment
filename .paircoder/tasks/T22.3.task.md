# Objective

Expose dispute letter generation via `POST /v1/disputes/letters`. Accepts negative items + bureau + letter type, returns generated letter(s). Supports batch generation for multiple items.

# Implementation Plan

1. Create `letter_routes.py` with router
2. Single letter endpoint: `POST /v1/disputes/letters`
   - Request body: LetterRequest
   - Response: GeneratedLetter
   - Auth required
   - Rate limited (30/minute)
3. Batch endpoint: `POST /v1/disputes/letters/batch`
   - Request body: list[LetterRequest] (max 10)
   - Response: list[GeneratedLetter]
   - Auth required
   - Rate limited (10/minute — heavier operation)
4. Register router in router.py under `/v1/disputes`
5. Add to OpenAPI schema with examples

# Acceptance Criteria

- [ ] POST /v1/disputes/letters generates single letter
- [ ] POST /v1/disputes/letters/batch generates multiple letters (max 10)
- [ ] Auth required on both endpoints
- [ ] Rate limited (30/min single, 10/min batch)
- [ ] Response includes full letter text, legal citations, bureau address
- [ ] Batch validates max 10 items
- [ ] Tests for single and batch endpoints
- [ ] Tests for auth, validation, rate limits
