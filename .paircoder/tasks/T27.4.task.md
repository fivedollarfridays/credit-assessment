# Objective

Fix 2 MEDIUM-severity findings: replace `'unsafe-inline'` in CSP style-src with hash-based directives, and document JWT role freshness architecture decision.

# Findings

| ID | File | Issue |
|----|------|-------|
| A05-2 | dashboard_routes.py:86 | `style-src 'unsafe-inline'` enables CSS-based data exfiltration |
| A01-1 | data_rights_routes.py:31-44 | JWT role-claim trusted for RBAC but stale after DB role downgrade |

# Implementation Plan

## A05-2: CSP style-src hash
1. Audit `dashboard.html` for all inline `<style>` blocks and `style=""` attributes
2. Compute SHA-256 hashes for each inline style
3. Replace `'unsafe-inline'` in `style-src` with specific `'sha256-...'` directives
4. Test: verify CSP header contains hash directives, no `'unsafe-inline'` in style-src

## A01-1: JWT role freshness
1. Document the architectural decision: JWT role claims are trusted for the token lifetime
2. Add inline code comment in `data_rights_routes.py` explaining the tradeoff
3. Consider adding `role_updated_at` timestamp check if token age > threshold (optional enhancement)
4. If implementing: add middleware that re-checks DB role for tokens older than N minutes
5. Test: if code change made, verify stale role detection works
