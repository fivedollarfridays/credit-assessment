# Objective

Implement role-based access control with three roles (admin, analyst, viewer) and scoped API keys per organization.

# Implementation Plan

1. Add Role enum and user-role association to database models
2. Create API key model: key, org_id, role, created_at, expires_at
3. Add role-checking dependency for FastAPI routes
4. Admin endpoints: manage users, view all org data
5. Analyst: run assessments, view own org data
6. Viewer: read-only access to assessment results
7. API key CRUD endpoints for admins
8. Write tests for role enforcement on all endpoints

# Acceptance Criteria

- [ ] Three roles enforced: admin, analyst, viewer
- [ ] API keys scoped to organization
- [ ] Unauthorized role access returns 403
- [ ] Admins can create/revoke API keys
- [ ] API keys have configurable expiration
- [ ] Tests pass locally (`pytest`)
