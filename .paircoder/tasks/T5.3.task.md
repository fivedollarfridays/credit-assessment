# Objective

Set up alerting for error spikes, latency degradation, and service health using PagerDuty or Opsgenie.

# Implementation Plan

1. Define alert rules: error rate > 5%, p95 latency > 500ms, health check fails
2. Create alerting configuration (Prometheus alertmanager or cloud-native)
3. Configure escalation policies and notification channels
4. Add alert test/silence endpoints for ops
5. Document alert runbooks

# Acceptance Criteria

- [ ] Alerts fire on error rate > 5%
- [ ] Alerts fire on p95 latency > 500ms
- [ ] Alerts fire on health check failure
- [ ] Escalation policies configured
- [ ] Alert runbooks documented
- [ ] Tests pass locally (`pytest`)
