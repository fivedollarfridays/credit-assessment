# Implementation Plan

## A06-1: Remove Unused `requests`

**Fix:** Remove `requests>=2.32.4,<3` from `[project.dependencies]`. Move to dev dependencies if needed for tests.

## A07-2: Demo Credentials from Environment

**Current:** `_DEMO_USERS = {"admin": "admin"}` hardcoded constant.
**Fix:** Read from `DEMO_USERNAME` / `DEMO_PASSWORD` env vars. If unset, demo auth disabled everywhere.

- `auth_routes.py`: Replace `_DEMO_USERS` dict with env var lookup in `_get_demo_users()`
- `config.py`: Add `demo_username: str | None = None` and `demo_password: str | None = None`
- Tests: Update demo auth tests

## A01-2: Assessment List Payload Redaction

**Current:** `GET /assessments` returns full `request_payload` and `response_payload`.
**Fix:** Return summary only: `score_band`, `barrier_severity`, `created_at`, `id`. Full payload via `GET /assessments/{id}`.

Note: assess_routes.py at 369 lines. This should replace inline logic, not add functions.

## A10-2: Deploy Health Check URL Validation

**Current:** `validate_health(base_url)` accepts any URL.
**Fix:** Validate scheme is `http` or `https`. Reject other schemes.

- `deploy.py`: Add `urllib.parse.urlparse` scheme check at top of `validate_health`
- Tests: Verify rejection of `file://`, `ftp://` schemes
