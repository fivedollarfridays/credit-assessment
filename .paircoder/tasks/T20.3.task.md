# Objective

Production deployment hardening: multi-worker process, Redis for distributed rate limiting, and OpenAPI docs accessible in production.

# Implementation Plan

1. **Dockerfile** — switch to gunicorn with uvicorn workers, configurable worker count via WEB_CONCURRENCY env var
2. **docker-compose.yml** — add Redis service, set REDIS_URL for API container
3. **OpenAPI docs** — enable /docs and /redoc behind auth in production (require admin role), or provide authenticated /v1/openapi.json endpoint
4. **Health check** — add Redis connectivity check to readiness probe
5. Update tests

# Acceptance Criteria

- [ ] Dockerfile uses gunicorn with configurable workers
- [ ] docker-compose.yml includes Redis service
- [ ] Rate limiting uses Redis when available
- [ ] OpenAPI spec accessible in production (behind auth)

- [ ] Tests pass
