# Objective

Create a template system for dispute letters. Define 5 letter types with legal citations, required fields, body structure, and 2-3 variations per type to avoid automation detection by bureaus.

# Implementation Plan

1. Create `letter_types.py` with `LetterType` enum:
   - VALIDATION: debt validation request (FDCPA 809)
   - INACCURACY: dispute inaccurate information (FCRA 611)
   - COMPLETENESS: dispute incomplete reporting (FCRA 623)
   - OBSOLETE_ITEM: demand removal of time-barred items (FCRA 605)
   - IDENTITY_THEFT: identity theft dispute (FCRA 605B)
2. Create `Bureau` enum: EQUIFAX, EXPERIAN, TRANSUNION
3. Create `LetterTemplate` dataclass/model:
   - letter_type: LetterType
   - variation: int (1-3)
   - subject_template: str
   - body_template: str (with placeholders: {consumer_name}, {creditor}, {account_number}, {amount}, {date}, {bureau_name}, {bureau_address})
   - legal_citations: list[str] (FCRA/FDCPA section references)
   - required_fields: list[str] (what must be provided to use this template)
4. Create `letter_templates.py` with template registry:
   - TEMPLATES: dict[LetterType, list[LetterTemplate]] (2-3 variations each)
   - get_template(letter_type, variation=None) -> LetterTemplate (random if variation not specified)
   - Bureau addresses: dict[Bureau, BureauAddress] with mailing addresses
5. Write all 5 x 3 = 15 template variations with legally accurate citations
