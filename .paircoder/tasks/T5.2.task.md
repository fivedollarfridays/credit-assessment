# Objective

Implement blue/green deployment strategy for zero-downtime releases with automated rollback.

# Implementation Plan

1. Create deployment scripts supporting blue/green swap
2. Use health check + readiness probe to validate new version before traffic switch
3. Add graceful shutdown handling (SIGTERM) to FastAPI app
4. Configure load balancer to route traffic to active deployment
5. Add rollback script that reverts to previous version
6. Document deployment runbook

# Acceptance Criteria

- [x] New version deployed without dropping requests
- [x] Health check validates new version before traffic switch
- [x] Graceful shutdown drains in-flight requests
- [x] Rollback reverts to previous version in < 30 seconds
- [x] Deployment documented in runbook
- [x] Tests pass locally (`pytest`)
