# Objective

Add PostgreSQL database with SQLAlchemy ORM and Alembic migrations for user accounts, assessment history, and audit logging.

# Implementation Plan

1. Add `sqlalchemy>=2.0`, `asyncpg`, `alembic` to dependencies
2. Create `src/modules/credit/database.py`: async engine, session factory
3. Create `src/modules/credit/models_db.py`: SQLAlchemy models (User, AssessmentRecord, AuditLog)
4. Initialize Alembic: `alembic init alembic`
5. Create initial migration with all tables
6. Add PostgreSQL service to docker-compose.yml
7. Create `src/modules/credit/repository.py`: data access layer
8. Write tests with SQLite in-memory for unit tests

# Acceptance Criteria

- [ ] PostgreSQL runs in docker-compose
- [ ] Alembic migrations create all tables
- [ ] `alembic upgrade head` works against running database
- [ ] Repository layer provides CRUD operations
- [ ] Assessment results can be persisted and retrieved
- [ ] Tests pass with SQLite in-memory backend
- [ ] Tests pass locally (`pytest`)
