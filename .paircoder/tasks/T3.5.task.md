# Objective

Integrate Sentry for automatic error tracking, performance monitoring, and release tracking.

# Implementation Plan

1. Add `sentry-sdk[fastapi]` to dependencies
2. Initialize Sentry in app startup with DSN from config
3. Configure: environment tagging, release version, sample rates
4. Add request ID as Sentry tag for correlation with structured logs
5. Test that exceptions are captured (mock Sentry in tests)

# Acceptance Criteria

- [ ] Sentry initializes on app startup when SENTRY_DSN is set
- [ ] Unhandled exceptions automatically reported to Sentry
- [ ] Request IDs included as Sentry tags
- [ ] Performance monitoring captures transaction traces
- [ ] No-op when SENTRY_DSN is not set (development mode)
- [ ] Tests pass locally (`pytest`)
