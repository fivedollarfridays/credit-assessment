# Objective

Fix two MEDIUM-severity findings: missing compliance audit entries on assessment endpoints (A09-2) and X-Forwarded-For spoofing risk (RL-1).

# Implementation Plan

## A09-2: Assessment Audit Trail

**Current:** `_run_assessment()` persists results but does not create an audit trail entry.
**Fix:** Add `create_audit_entry()` call as a background task with `action="assessment_run"` and summary containing `score_band` + `barrier_severity` (no PII).

Note: assess_routes.py is at 369 lines / 13 functions. Add 2-3 lines to existing background task section only.

- `assess_routes.py`: Add audit entry in `_run_assessment()` background tasks
- Tests: Verify audit entry is created after assessment

## RL-1: Rate Limit Proxy Trust Configuration

**Current:** Docstring documents the risk but no `FORWARDED_ALLOW_IPS` in compose.
**Fix:** Add `FORWARDED_ALLOW_IPS` env var to docker-compose files. Add `trusted_proxy_ips` setting to config.

- `docker-compose.yml`: Add `FORWARDED_ALLOW_IPS` env var
- `docker-compose.deploy.yml`: Add `FORWARDED_ALLOW_IPS` with `:?` required syntax
- `config.py`: Add `trusted_proxy_ips: str | None = None` setting
- Tests: Verify setting is loaded
