# Objective

Add account lockout after repeated failed logins. Password reset token expiry is handled in T18.1 — this task adds the lockout mechanism.

# Implementation Plan

1. Track failed login attempts per email (count + last_attempt timestamp)
2. Lock account after 5 consecutive failures (configurable)
3. Auto-unlock after 15 minutes (configurable)
4. Return 429 (Too Many Requests) when locked, with Retry-After header
5. Reset counter on successful login
6. Log lockout events to audit trail

# Acceptance Criteria

- [x] Account locks after 5 consecutive failures
