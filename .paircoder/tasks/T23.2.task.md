# Objective

Expose dispute lifecycle management via REST endpoints. Create, list, and update disputes. Auto-calculate FCRA 30/45-day deadlines on status transitions.

# Implementation Plan

1. Create `dispute_routes.py` with router
2. Endpoints:
   - `POST /v1/disputes` — create dispute from NegativeItem + bureau
     - Auto-sets status=DRAFT
     - Auth required, user_id from auth identity
   - `GET /v1/disputes` — list user's disputes
     - Query params: status (filter), limit, offset
     - Returns paginated list with total count
     - Scoped to authenticated user (admin can see org-wide)
   - `GET /v1/disputes/{dispute_id}` — get single dispute
     - User ownership check (or admin)
   - `PATCH /v1/disputes/{dispute_id}/status` — update dispute status
     - Request body: { status: DisputeStatus, resolution?: str }
     - Validates transition (e.g. DRAFT -> SENT allowed, DRAFT -> RESOLVED not)
     - Auto-sets sent_at, deadline_at, responded_at as appropriate
   - `GET /v1/disputes/deadlines` — list approaching deadlines
     - Query param: days_ahead (default 7)
     - Returns disputes with deadlines within N days
3. Register router in router.py under `/v1/disputes`
4. Rate limit: 30/minute on create, 60/minute on reads
