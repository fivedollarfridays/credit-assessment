# Objective

Create Alembic migration infrastructure and generate the initial migration from the expanded schema.

# Implementation Plan

1. Run `alembic init alembic` to create migration directory structure
2. Configure `alembic/env.py` to use async engine and import Base metadata
3. Configure `alembic.ini` to use DATABASE_URL from settings
4. Generate initial migration: `alembic revision --autogenerate -m "initial schema"`
5. Verify migration applies cleanly: `alembic upgrade head`
6. Replace `Base.metadata.create_all` in router.py startup with Alembic
7. Test downgrade path works

# Acceptance Criteria

- [ ] alembic/ directory with env.py and versions/ exists
- [ ] Initial migration script generated from all ORM models
- [ ] `alembic upgrade head` succeeds on fresh database
- [ ] `alembic downgrade base` succeeds
- [ ] router.py startup uses Alembic instead of create_all
- [ ] Tests pass
- [ ] 0 ruff issues

# Verification

- `alembic upgrade head && alembic downgrade base && alembic upgrade head`
- `python -m pytest src/ -q`
