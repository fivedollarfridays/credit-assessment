# Objective

Fix 2 MEDIUM-severity findings: replace SHA-256 KDF with PBKDF2HMAC, and add PII scrubbing to Sentry and structlog.

# Findings

| ID | File | Issue |
|----|------|-------|
| A02-2 | crypto.py:16-19 | SHA-256 as KDF — no salt, no iterations, brute-forceable |
| A09-1 | sentry.py + logging_config.py | No PII scrubbing in structlog or Sentry `before_send` |

# Implementation Plan

## A02-2: PBKDF2HMAC KDF
1. Replace `hashlib.sha256(passphrase.encode()).digest()` with `PBKDF2HMAC(SHA256, 32, salt, 600_000)`
2. Use a configurable salt (env var `WEBHOOK_ENCRYPTION_SALT` or derive from key)
3. Maintain backward compatibility: try new KDF first, fall back to old SHA-256 for existing ciphertexts
4. Update `lru_cache` to include salt in cache key
5. Test: verify encryption/decryption works with new KDF, old ciphertexts still decrypt

## A09-1: PII scrubbing
1. In `sentry.py`, add `before_send` hook that scrubs email, JWT, API key, and credit data from events
2. In `logging_config.py`, add structlog processor that redacts PII patterns (email, SSN, card numbers)
3. Test: verify PII is scrubbed from sample Sentry events and log entries
