# Objective

Containerize the Credit Assessment API with a production-ready Dockerfile and a docker-compose setup for local development.

# Files to Create

| File | Purpose |
|------|---------|
| `Dockerfile` | Multi-stage build: deps, test, production |
| `docker-compose.yml` | Local dev environment with hot reload |
| `.dockerignore` | Exclude .git, .venv, __pycache__, etc. |
| `src/main.py` | Uvicorn entry point for the container |

# Implementation Plan

1. Create `src/main.py` entry point that imports `app` from `modules.credit.router`
2. Create multi-stage `Dockerfile`:
   - **Stage 1 (builder):** Python 3.13-slim, install deps from pyproject.toml
   - **Stage 2 (test):** Run pytest to verify build
   - **Stage 3 (production):** Minimal image, non-root user, expose 8000
3. Create `.dockerignore` to exclude dev files
4. Create `docker-compose.yml` with:
   - `api` service: build from Dockerfile, port 8000, env vars, volume mount for dev
   - Health check using `/health` endpoint
5. Test: `docker build -t credit-api .` and `docker-compose up`

# Acceptance Criteria

- [x] `docker build -t credit-api .` succeeds
- [x] `docker run -p 8000:8000 credit-api` serves `/health` endpoint
- [x] `docker-compose up` starts the API with hot reload
- [x] Container runs as non-root user
- [x] Image size < 200MB (slim base)
- [x] All 196 tests pass inside the container
- [x] Tests pass locally (`pytest`)
