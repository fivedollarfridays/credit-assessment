# Objective

Create load testing scripts to validate API throughput, latency percentiles, and error rates under sustained load.

# Implementation Plan

1. Create `loadtests/` directory with Locust scripts
2. Define scenarios: health check, single assessment, burst assessments
3. Configure target thresholds: p95 < 200ms, p99 < 500ms, 0% errors at 100 RPS
4. Add locust to dev dependencies
5. Create docker-compose override for running load tests
6. Document how to run and interpret results

# Acceptance Criteria

- [ ] Locust script covers /health and /assess endpoints
- [ ] Baseline performance report generated
- [ ] p95 latency < 200ms for /assess at 50 RPS
- [ ] 0% error rate at sustained load
- [ ] Load test runnable via docker-compose
- [ ] Tests pass locally (`pytest`)
