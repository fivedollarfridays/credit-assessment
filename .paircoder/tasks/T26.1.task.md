# Objective

Fix two HIGH-severity audit findings: raw API key exposure in URL path (A01-1) and silent DATABASE_URL password default (A02-1).

# Findings

| ID | Severity | File | Issue |
|----|----------|------|-------|
| A01-1 | HIGH | admin_routes.py:96 | Raw API key in DELETE URL path — appears in server logs, proxies, Sentry |
| A02-1 | HIGH | docker-compose.yml:39 | DATABASE_URL has `:-credit` fallback — silently uses default password |

# Implementation Plan

## A01-1: API Key Revocation Route

**Current:** `DELETE /admin/api-keys/{api_key}` accepts raw secret as path param.
**Fix:** Change to `DELETE /admin/api-keys/{key_prefix}` using the 8-char `key_prefix` already stored in `ApiKeyDB`.

- `admin_routes.py`: Change route param from `{api_key}` to `{key_prefix}`, call `repo.revoke_by_prefix(key_prefix)`
- `repo_api_keys.py`: Add `revoke_by_prefix(prefix: str)` that looks up by `key_prefix` column
- Tests: Update admin route tests to use prefix instead of raw key

## A02-1: DATABASE_URL Required Password

**Current:** `${POSTGRES_PASSWORD:-credit}` silently defaults to `credit` in api service DATABASE_URL.
**Fix:** Change to `${POSTGRES_PASSWORD:?POSTGRES_PASSWORD must be set}`.

- `docker-compose.yml`: Change `:-credit` to `:?POSTGRES_PASSWORD must be set` in DATABASE_URL

# Acceptance Criteria

- [x] DELETE endpoint uses `key_prefix` parameter, not raw API key
- [x] Raw API key never appears in any URL
- [x] `revoke_by_prefix()` finds and revokes by the 8-char prefix
- [x] DATABASE_URL fails fast if POSTGRES_PASSWORD not set
- [x] Existing admin route tests updated and passing
