# Objective

Replace the global 30/min rate limit with tiered per-customer limits based on subscription plan.

# Implementation Plan

1. Define rate limit tiers: free (10/min), starter (60/min), pro (300/min), enterprise (unlimited)
2. Create rate limit resolver that looks up customer's plan
3. Replace global slowapi limiter with per-key dynamic limits
4. Add rate limit headers: X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset
5. Redis backend for distributed rate limiting (add to docker-compose)
6. Write tests for each tier and limit exhaustion

# Acceptance Criteria

- [ ] Rate limits vary by customer subscription tier
- [ ] Rate limit headers included in all responses
- [ ] 429 response includes retry-after information
- [ ] Redis-backed for multi-instance deployments
- [ ] Graceful fallback if Redis unavailable
- [ ] Tests pass locally (`pytest`)
