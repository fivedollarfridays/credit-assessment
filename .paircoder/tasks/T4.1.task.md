# Objective

Implement user registration, login, and password reset flows. Option to delegate to Auth0/Clerk or build in-house with the existing JWT infrastructure.

# Implementation Plan

1. Create user model with hashed passwords (bcrypt)
2. Add endpoints: POST /auth/register, POST /auth/login, POST /auth/reset-password
3. Email verification flow (or skip for MVP with manual activation)
4. Rate limit auth endpoints to prevent brute force
5. Write tests for registration, login, invalid credentials, password reset

# Acceptance Criteria

- [ ] Users can register with email + password
- [ ] Login returns JWT access + refresh tokens
- [ ] Password reset flow works (token-based)
- [ ] Passwords stored as bcrypt hashes, never plaintext
- [ ] Auth endpoints rate-limited
- [ ] Tests pass locally (`pytest`)
