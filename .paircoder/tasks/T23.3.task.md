# Objective

Add dedicated score history tracking with source attribution and trend analysis. Auto-record scores on each assessment and expose a timeline endpoint for longitudinal monitoring.

# Implementation Plan

1. Create `ScoreSource` enum: ASSESSMENT, MANUAL, EXTERNAL
2. Create `ScoreHistory` ORM model in models_db.py:
   - id: primary key
   - user_id: str (indexed)
   - org_id: str | None (tenant isolation)
   - score: int (300-850)
   - score_band: str
   - source: str (ScoreSource value)
   - assessment_id: int | None (FK to AssessmentRecord if source=ASSESSMENT)
   - notes: str | None
   - recorded_at: datetime (auto)
3. Create `ScoreHistoryRepository` in repo_scores.py:
   - record(user_id, score, source, assessment_id?, notes?) -> ScoreHistory
   - list_by_user(user_id, limit, offset) -> list[ScoreHistory]
   - get_latest(user_id) -> ScoreHistory | None
   - get_trend(user_id, days=90) -> list[ScoreHistory]
4. Hook into _run_assessment() in assess_routes.py:
   - After successful assessment, if user is authenticated (not API key), auto-record score
5. Create `GET /v1/scores/history` endpoint:
   - Query params: limit, offset, days (time window)
   - Response includes: entries, latest score, delta from previous, trend direction (up/down/stable)
6. Create `POST /v1/scores` endpoint for manual score entry:
   - Body: { score: int, notes?: str }
   - Source = MANUAL
7. Alembic migration for score_history table
