# Objective

Integrate Stripe for subscription management and per-assessment usage metering.

# Implementation Plan

1. Add `stripe` SDK to dependencies
2. Create billing module: subscription plans, usage records, webhook handler
3. Define plans: Free, Starter ($29/mo), Pro ($99/mo), Enterprise (custom)
4. Meter each /assess call as a usage event
5. Stripe webhook endpoint for subscription lifecycle events
6. Create billing portal redirect endpoint
7. Write tests with Stripe test mode / mocked responses

# Acceptance Criteria

- [ ] Stripe checkout creates subscriptions
- [ ] Each /assess call metered as usage event
- [ ] Stripe webhooks update local subscription status
- [ ] Billing portal accessible for self-service management
- [ ] Subscription changes reflected in rate limits immediately
- [ ] Tests pass with mocked Stripe API
- [ ] Tests pass locally (`pytest`)
