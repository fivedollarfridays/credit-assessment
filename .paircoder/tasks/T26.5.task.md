# Objective

Fix five INFO-severity audit findings in a single batch.

# Findings

| ID | Severity | File | Issue |
|----|----------|------|-------|
| D-2 | INFO | docker-compose.yml:41 | Dev volume mount may persist into prod |
| CI-1 | INFO | docker-publish.yml | Actions pinned to major tags, not SHAs |
| A06-2 | INFO | pyproject.toml:17 | Tight cryptography version pin |
| RC-1 | INFO | deploy.py:14 | Shutdown flag not formally thread-safe |
| A08-2 | INFO | .github/workflows/ | Duplicate of CI-1 |

# Implementation Plan

## D-2: Prod Volume Override

**Fix:** Add `volumes: []` override in `docker-compose.deploy.yml` for the api service.

## CI-1 / A08-2: Pin Actions to SHAs

**Fix:** Replace `@v4`/`@v5` tags with full commit SHAs in all workflow files. Add version comment for readability.

Files: `ci.yml`, `docker-publish.yml`, `sdk-ci.yml`

## A06-2: Widen Cryptography Pin

**Current:** `cryptography>=46.0.5,<47`
**Fix:** Widen to `cryptography>=46.0.5,<48`.

## RC-1: Thread-Safe Shutdown Flag

**Current:** `_shutting_down = False` boolean.
**Fix:** Replace with `threading.Event()`. Use `_shutdown_event.is_set()` and `_shutdown_event.set()`.
