# Objective

Create a score impact simulation engine that answers "what if I do X?" questions. Given a current CreditProfile and a list of proposed actions, compute projected score changes, new readiness, and updated eligibility. Pure computation — no DB or endpoints.

# Implementation Plan

1. Create `simulation.py` module with `ScoreSimulator` class
2. Define `SimulationAction` Pydantic model:
   - action_type: ActionType (from existing enum)
   - target_amount: float | None (e.g. pay down $2000)
   - target_item: str | NegativeItem | None (e.g. which collection to remove)
3. Define `SimulationResult` Pydantic model:
   - original_score: int
   - projected_score: int
   - score_delta: ScoreImpact (min/max/expected)
   - projected_readiness: CreditReadiness
   - projected_eligibility: list[EligibilityItem]
   - actions_applied: list[SimulationAction]
4. Implement action handlers:
   - PAY_DOWN_DEBT: reduce balance -> recalculate utilization -> score impact from utilization brackets
   - REMOVE_COLLECTION: remove from negative items -> score boost (~20-50 pts based on recency)
   - REDUCE_UTILIZATION: direct utilization change -> bracket-based impact
   - PAY_ON_TIME: improve payment_history_pct -> weighted impact
   - BECOME_AUTHORIZED_USER: add account age -> credit age factor improvement
   - OPEN_SECURED_CARD: improve credit mix -> small positive
   - KEEP_ACCOUNTS_OPEN: prevent credit age reduction
   - AVOID_NEW_INQUIRIES: prevent new credit penalty
   - DISPUTE_NEGATIVE: remove negative item -> same as REMOVE_COLLECTION
   - DIVERSIFY_CREDIT_MIX: improve credit mix factor
5. Each handler modifies a copy of CreditProfile, then re-runs CreditAssessmentService.assess() to get new results
6. Stack multiple actions (apply sequentially, compound effects)
