# Objective

Implement feature flags for gradual rollouts, A/B testing, and kill switches using LaunchDarkly, Unleash, or a simple in-house solution.

# Implementation Plan

1. Create feature flag service with simple JSON/DB-backed store
2. Add flag evaluation to request middleware
3. Support targeting: by org, by user, by percentage
4. Create admin endpoints for flag management
5. Add flag evaluation to assessment pipeline for new features
6. Write tests for flag evaluation logic

# Acceptance Criteria

- [ ] Feature flags toggleable without deployment
- [ ] Targeting by org, user, and percentage supported
- [ ] Admin API for flag management
- [ ] Flag evaluation adds < 1ms latency
- [ ] Tests pass locally (`pytest`)
